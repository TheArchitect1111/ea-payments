# Demo Library

This library defines the examples needed for sales, onboarding, support, and product QA.

## Demo Library Goals

- Show what each product does without requiring private client data.
- Give operators backup links when a live capture is not appropriate.
- Cover the major Magnifi template categories.
- Provide examples for Simplifi capture, Magnifi story, Amplifi share, and Simplifi Guidance.

## Demo Record Checklist

Each demo should have:

- Capture Record in Airtable.
- Capture title.
- Business or organization name.
- Source URL or safe placeholder source.
- Blueprint summary.
- Recommendation summary.
- Opportunity score.
- EA fit score.
- Consider slug.
- Share URL.
- Magnifi URL.
- Simplifi guidance URL.
- One approved social/email share draft.

## Demo Review Checklist

- Opens without login when intended.
- Does not expose private information.
- Shows a clear opportunity.
- Has a strong first screen.
- Has a practical next action.
- Works on mobile and desktop.
- Can be explained in less than 30 seconds.

## Demo Naming

Use readable, stable slugs:

- `selena`
- `small-business-visibility`
- `community-blueprint`
- `university-ecosystem`
- `hidden-asset`

Avoid temporary test slugs in sales materials.

## Demo Maintenance

Review demos monthly:

- Confirm links still open.
- Confirm screenshots/copy match current product.
- Retire outdated demos.
- Add one new demo when a new vertical becomes important.
